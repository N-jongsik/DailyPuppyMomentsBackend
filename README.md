## 🔥 주요 기능

- **React**: 컴포넌트 기반 UI 설계로 빠르고 유연한 사용자 인터페이스 구현
- **Redux**: 강력한 상태 관리로 쇼핑몰 기능을 효과적으로 처리(상품 관리 기능, 장바구니 기능 등)
- **Kakao Map API**: 지도 기반 매장 검색 및 위치 확인 기능 탑재
- **Lazy Loading**: 성능 최적화를 위한 지연 로딩으로 더 빠르고 부드러운 사용자 경험 제공

---

## 🎯 구현 내용

- **반응형 디자인**: 모든 디바이스에서 완벽하게 작동하는 레이아웃
- **상품 페이지**: Redux를 이용하여 상품 리스트 및 상품 상세 정보를 포함한 동적인 상품 표시
- **장바구니 기능**: Redux를 이용한 장바구니 담기, 업데이트 및 상태 유지
- **지도 연동**: Kakao Map API로 매장 위치 검색

---

## 🚀 기술 스택

- **프론트엔드**: React, Tailwind CSS, Redux
- **백엔드**: SpringBoot, JPA, MySql, SpringSecurity, Oauth
- **로그인 API**: Kakao Login API

## 🎤 소감

사전 기획 단계에서 목표한 기능은 대부분 달성. 목표 달성도 측면에서는 우수하나 완성도 측면에서는 아쉬움 <br>
소셜 로그인이나 특정 라이브러리 등 자주 사용하지 못했던 기술 이용으로 어려움을 겪었음. 하지만 기획단계에서 설계한 기능은 모두 완성함.<br>
사용자에게 제공되는 사용자 경험을 고려하여 프로젝트를 구상하는 것이 가장 중요하다고 느낌. 프로젝트의 3요소를 적절히 정하여 테마의 맞는 기능을 구현한 성과를 경험함

# commit message

feat : 새로운 기능 추가

fix : 버그 수정

docs : 문서 수정

style : 코드 스타일 변경 (코드 포매팅, 세미콜론 누락 등) 기능 수정이 없는 경우

design : 사용자 UI 디자인 변경 (CSS 등)

test : 테스트 코드, 리팩토링 테스트 코드 추가

refactor : 코드 리팩토링

build : 빌드 파일 수정

ci : CI 설정 파일 수정

perf : 성능 개선

chore : 빌드 업무 수정, 패키지 매니저 수정 (gitignore 수정 등)

rename : 파일 혹은 폴더명을 수정만 한 경우

remove : 파일을 삭제만 한 경우
